# Minecraft コマンド学習アプリ 要件定義

## 概要

Minecraftのコマンドを子どもが楽しく学べるウェブアプリ。
**Java Edition 専用**であることをトップ画面に明記する。

---

## ターゲットユーザー

- 年齢：小学生〜中学生（8〜14歳）
- Minecraftをすでにプレイしている、またはプレイしたことがある子ども
- 保護者が子どもの学習をサポートする場面にも対応

---

## 解決したい課題

- Minecraftのコマンドは覚えることが多く、どこから始めればよいか分からない
- 公式ドキュメントは英語・専門用語が多く子どもには難しい
- 実際に打ち込んでみるまで結果がイメージしにくい

---

## 主要機能

### クイズ機能
- トップ画面で難しさを選ぶ → ランダムに出題 → 回答、を繰り返す
- 問題・選択肢はJSONファイルで定義し、そこからランダムに出題する
- 1問ずつ回答 → 即時に正解/不正解のフィードバックを表示
- セッション内のスコア（正解数 / 出題数）を表示
- ログイン・進捗保存は不要（完全に読み取り専用アプリ）

#### 難しさの3ステージ

| 難しさ | お題 | 選択肢の構成 |
|---|---|---|
| かんたん | 「〜することができるコマンドは？」 | 全く異なるコマンドが3つ並ぶ（例：`/give` vs `/tp` vs `/weather`） |
| ふつう | 同上 | スペルが似ているが異なるコマンドが並ぶ（例：`/effect` vs `/affect` vs `/efffect`） |
| むずかしい | 同上 | コマンド名は同じで、引数・構文が異なるバリエーションが並ぶ |

## 対象コマンド（カテゴリ案）

| カテゴリ | 例 |
|---|---|
| テレポート | `/tp`、`/teleport` |
| アイテム | `/give`、`/clear` |
| 時間・天気 | `/time set`、`/weather` |
| ゲームモード | `/gamemode` |
| エフェクト | `/effect` |
| ブロック操作 | `/fill`、`/setblock`、`/clone` |
| エンティティ | `/summon`、`/kill` |
| スコアボード | `/scoreboard` |
| メッセージ | `/say`、`/tell`、`/title` |

---

## 非機能要件

| 項目 | 内容 |
|---|---|
| 対応デバイス | **タブレット優先**（PC・スマートフォンも対応、レスポンシブ） |
| 対象バージョン | **Java Edition 専用**（トップ画面に明記） |
| 認証 | 不要（ログインなし・完全読み取り専用） |
| 言語 | 日本語（UIテキスト・JSONは難しい漢字・英語を避け、平易な日本語で記述） |
| UIタップ操作 | タップしやすい十分なタップターゲットサイズ（最小44px） |
| アクセシビリティ | フォントサイズ大きめ・カラーコントラストに配慮 |

## デザイン方針

- **Minecraftライクなビジュアルテーマ**
  - ドット絵・ピクセルフォント（Minecraft標準フォントの雰囲気）
  - 土・草・石・木材などMinecraftのブロックを彷彿とさせる配色・テクスチャ
  - ボタンやカードのデザインをMinecraftのUIに寄せる
- **子ども向けUX**
  - ボタン・カードを大きく取り、誤タップを防ぐ余白を確保
  - UIテキストは難しい漢字・英語を避け、平易な日本語を使用
- **サウンド・アニメーション**
  - 正解・不正解・選択などの操作に対してサウンドを積極的に活用
  - 画面遷移・回答フィードバックなどにアニメーションを多めに取り入れ、楽しい体験を演出

## データ構造（JSONファイル）

クイズ問題は `/data/questions.json`（または難しさ別にファイル分割）で管理。
UIに表示するテキストは難しい漢字・英語を避け、平易な日本語で記述する。
選択肢（`choices`）はJSONには持たず、難しさのルールに従って動的に生成する。

```json
{
  "difficulty": "easy",
  "questions": [
    {
      "id": "q001",
      "description": "プレイヤーをべつの場所に移動させるコマンドは？",
      "command": "/tp",
      "args": [
        { "name": "たいしょう", "type": "プレイヤーめい", "required": true },
        { "name": "いどう先のX", "type": "すうじ", "required": false },
        { "name": "いどう先のY", "type": "すうじ", "required": false },
        { "name": "いどう先のZ", "type": "すうじ", "required": false }
      ]
    }
  ]
}
```

### `args` の各フィールド

| フィールド | 型 | 説明 |
|---|---|---|
| `name` | string | 引数の名前（平易な日本語） |
| `type` | string | 引数の型・種類（例：プレイヤーめい、すうじ、true/false など） |
| `required` | boolean | 省略できない引数かどうか |

## コマンドデータのコンバート仕様

### データソース

[misode/mcmeta](https://github.com/misode/mcmeta) が公開している `summary/commands/data.json` を使用する。
このファイルはMinecraftサーバーの `--reports` で生成されるBrigadierコマンドツリーを整形したもの。

### コンバーターの仕様

- **入力**：`misode/mcmeta` の `summary/commands/data.json`（GitHubから直接fetchまたはローカルに配置）
- **出力**：アプリ用の `src/data/commands.json`
- **実装場所**：`tools/convert/` ディレクトリ（Node.js + TypeScript）

#### 変換ルール

| 入力（Brigadierツリー） | 出力（アプリ用JSON） |
|---|---|
| コマンド名（ルートノード） | `command` フィールド |
| 引数ノード（argument type） | `args[]` の各要素 |
| 引数名 | `args[].name`（英語のまま。後で日本語に手動置換） |
| 引数の型（例：`minecraft:entity`） | `args[].type`（英語のまま） |
| 必須 / 省略可（`executable` フラグ等から推定） | `args[].required` |

#### 出力JSONのスキーマ

```json
[
  {
    "id": "tp",
    "command": "/tp",
    "description": "",
    "args": [
      { "name": "destination", "type": "minecraft:entity", "required": true }
    ]
  }
]
```

- `description` は空文字で出力し、後から手動で日本語を追記する
- 引数の `name` と `type` も後から日本語に手動置換する
- コマンドの絞り込み（学習対象外のコマンドの除外）はコンバーター実行後に手動で行う

### コンバーターの実行方法

```bash
cd tools/convert
npm install
npm run convert
# → src/data/commands.json が生成される
```
